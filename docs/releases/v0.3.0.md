# v0.3.0 Stable Release

This is the first stable package release for `mcp-server-pronunciation`.

Pronunciation feedback remains a coaching signal, not a standardized-test,
clinical, employment, or high-stakes assessment. Review transcripts and
feedback carefully, especially with noisy audio, short clips, rare names,
domain terms, or strong accent variation.

## Highlights

- Stable PyPI and MCP Registry metadata for `0.3.0`.
- Local voice conversation with English pronunciation, grammar, and fluency
  feedback.
- Drill mode with word alignment, phoneme feedback, learner-profile hints,
  minimal-pair drills, and prosody checks.
- Optional `[phoneme]` extra for wav2vec2 forced alignment to reduce some
  Whisper-bias false positives on reference sentences.
- Improved `doctor` configuration checks for model selection, preload behavior,
  and temporary recording retention.
- More actionable recording errors for missing input devices, microphone
  permissions, PortAudio issues, empty captures, and WSL PowerShell timeouts.
- MCP smoke tests for tool discovery and text-only tool calls without audio
  hardware.

## Install

```bash
uvx mcp-server-pronunciation@0.3.0
```

For pip users:

```bash
pip install mcp-server-pronunciation==0.3.0
```

Run the preflight before wiring it into an MCP client:

```bash
mcp-server-pronunciation doctor
```
